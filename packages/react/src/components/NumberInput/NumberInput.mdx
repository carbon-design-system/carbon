## Number formatting

Since there are a number of limitations and ux issues surrounding
`input type="number"`, the input `type` is configurable to be `text` instead.
When using `type="text"`, provide `pattern` and `inputmode` attributes based on
the numbers you intend to accept.

Number parsing and formatting helpers available through `@carbon/utilities` can
be used with `onChange`/`onBlur` to support locale formatting.
