## Overview

The Composable FileUploader is a flexible, user-controlled alternative to the
monolithic FileUploader component. It follows the compound component pattern,
providing better alignment with the Web Components implementation and greater
flexibility for developers.

### Key Features

- **User-controlled state**: You manage the file list, giving you full control
- **Composable**: Mix and match Button, DropContainer, List, and Item components
- **Better error handling**: Display custom error messages for invalid files
- **No feature flags**: All functionality is available without experimental
  flags
- **Web Components parity**: Matches the architecture of `cds-file-uploader`

### When to Use

Use the Composable FileUploader when you need:

- Custom file validation logic
- Custom error message display
- Multiple input methods (button + drag-and-drop)
- Full control over file list rendering
- Integration with external state management

## Architecture

### Comparison with Monolithic FileUploader

| Feature                | Monolithic                     | Composable                     |
| ---------------------- | ------------------------------ | ------------------------------ |
| State management       | Internal                       | User-controlled                |
| File list rendering    | Fixed                          | Customizable                   |
| Invalid file handling  | Filtered out                   | User displays                  |
| Input methods          | Button only                    | Button, DropContainer, or both |
| Web Components parity  | ❌ No                          | ✅ Yes                         |
| Feature flags required | ✅ Yes (for enhanced features) | ❌ No                          |

### Pattern

The Composable FileUploader follows the compound component pattern:

```jsx
<FileUploader>
  {' '}
  {/* Container with context */}
  <FileUploader.Button /> {/* Input method */}
  <FileUploader.List>
    {' '}
    {/* File list container */}
    <FileUploader.Item /> {/* Individual file */}
  </FileUploader.List>
</FileUploader>
```

This matches the Web Components architecture:

```html
<cds-file-uploader>
  <cds-file-uploader-button slot="drop-container" />
  <cds-file-uploader-item />
</cds-file-uploader>
```

## Usage

### Basic Example with Button

<Canvas of={FileUploaderComposableStories.WithButton} />

```jsx
import { FileUploader } from '@carbon/react/lib/components/FileUploader/index.composable';
import { useState } from 'react';

function MyComponent() {
  const [files, setFiles] = useState([]);

  const handleAddFiles = (evt, { addedFiles }) => {
    const newFiles = addedFiles.map((file) => ({
      uuid: generateUuid(),
      name: file.name,
      status: 'edit',
      invalid: file.size > 500000,
    }));
    setFiles((prev) => [...prev, ...newFiles]);
  };

  const handleDelete = (evt, { uuid }) => {
    setFiles((prev) => prev.filter((f) => f.uuid !== uuid));
  };

  return (
    <FileUploader labelTitle="Upload files" labelDescription="Max 500kb">
      <FileUploader.Button onAddFiles={handleAddFiles} multiple>
        Add files
      </FileUploader.Button>
      <FileUploader.List>
        {files.map((file) => (
          <FileUploader.Item
            key={file.uuid}
            {...file}
            onDelete={handleDelete}
          />
        ))}
      </FileUploader.List>
    </FileUploader>
  );
}
```

### Custom Validation

```jsx
const handleAddFiles = (evt, { addedFiles }) => {
  const newFiles = addedFiles.map((file) => {
    const errors = [];

    // Custom validation
    if (file.size > 500000) {
      errors.push('File size exceeds 500kb');
    }
    if (!file.name.match(/\.(jpg|png)$/i)) {
      errors.push('Only JPG and PNG files are allowed');
    }

    return {
      uuid: generateUuid(),
      name: file.name,
      status: 'edit',
      invalid: errors.length > 0,
      errorSubject: errors.length > 0 ? 'Invalid file' : undefined,
      errorBody: errors.join('. '),
    };
  });

  setFiles((prev) => [...prev, ...newFiles]);
};
```
