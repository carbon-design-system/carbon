## Overview

Page footer presents the actions for a page-level flow and remains sticky at the
bottom of its nearest scrolling container. It uses the background and subtle
border theme tokens to remain distinct from scrolling page content.

## Usage

Import the preview component and provide up to four Carbon buttons as direct
children. Buttons inherit the 80px `2xl` layout size. A ghost button is placed
16px from the inline start while the remaining actions align to the inline end.
At narrow widths, the actions stack using the responsive behavior from
`ButtonSet`.

```jsx
import { Button, preview_PageFooter as PageFooter } from '@carbon/react';

<PageFooter aria-label="Creation flow actions">
  <Button kind="ghost">Cancel</Button>
  <Button kind="secondary">Previous</Button>
  <Button kind="primary">Next</Button>
</PageFooter>;
```

Avoid setting an explicit `size` on the child buttons unless the design calls
for a deliberate exception, because an explicit size overrides the footer's
`2xl` layout context.

## Accessibility

Use concise, localized labels that describe each action. Page footer renders an
ARIA `group`; provide an `aria-label` that identifies the group when the page
contains other sets of actions.
