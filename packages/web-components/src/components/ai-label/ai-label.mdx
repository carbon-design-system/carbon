The AI Label is intended for any scenario where something is being generated by
(or with) AI to reinforce AI transparency, accountability, and explainability at
any interface level. This also enables more effective recognition and recall of
AI instances in a way that is identifiable across any IBM product.

## Getting started

Here's a quick example to get you started.

The `AI Label` itself is a button that acts as a `Toggletip` trigger. To
construct the contents of the `AI Label` callout, you can place the desired
elements as a child of `AI Label` in a div with slot `body-text`, like so:

### JS (via import)

```javascript
import '@carbon/web-components/es/components/ai-label/index.js';
```

<Markdown>{`${cdnCss()}`}</Markdown>

### HTML

```html
<cds-ai-label size="mini" alignment="bottom-left">
  <div slot="body-text">{Content Here}</div>
</cds-ai-label>
```

### HTML (with actions)

The `AI Label` also supports the rendering of an action bar at the bottom of the
callout. To achieve this, you can add elements with `slot="actions"` inside the
`slot="body-text"` element

```html
<cds-ai-label size="mini" alignment="bottom-left">
  <div slot="body-text">
    {Content Here} {Optional AI Label action bar}
    <cds-icon-button kind="ghost" slot="actions" size="lg">
      ${View16({ slot: 'icon' })}
      <span slot="tooltip-content"> View </span>
    </cds-icon-button>
    <cds-icon-button kind="ghost" slot="actions" size="lg">
      ${FolderOpen16({ slot: 'icon' })}
      <span slot="tooltip-content"> Open folder</span>
    </cds-icon-button>
    <cds-icon-button kind="ghost" slot="actions" size="lg">
      ${Folders16({ slot: 'icon' })}
      <span slot="tooltip-content"> Folders </span>
    </cds-icon-button>
    <cds-ai-label-action-button>View details</cds-ai-label-action-button>
  </div>
</cds-ai-label>
```

### AI Label `aiText`

If needed, you can change the text inside the `AI Label` when translating to
different languages.

```html
<cds-ai-label ai-text="IA">
  <div slot="body-text">Explanation of AI-generated content</div>
</cds-ai-label>
```

### AI Label `aiTextLabel`

When using the `inline` variant, you can add text adjacent to the AI label with
the `ai-text-label` prop.

```html
<cds-ai-label kind="inline" ai-text-label="Text goes here">
  <div slot="body-text">Explanation of AI-generated content</div>
</cds-ai-label>
```

### AI Label `kind`

The `AI Label` component has two variants, `default` and `inline`.

```html
<cds-ai-label size="xs">
  <div slot="body-text">AI was used to generate this content</div>
</cds-ai-label>
```

The `inline` `AI Label` with the standard icon can also trigger the AI
explainability callout.

```html
<cds-ai-label kind="inline" ai-text-label="Text goes here" size="xs">
  <div slot="body-text">Explanation of AI-generated content</div>
</cds-ai-label>
```

### AI Label `revertActive`

`revert-active` is a boolean prop that can be set on `AI Label` that transforms
the `AI Label` into a revert action button. This is useful if a user edits an
AI-generated input to show that the element has been modified. This can be used
in conjunction with the `@click` event listener to handle restoring the element
to the AI-generated state. For an example, please take a look at the
[Form Group With AI Label](https://web-components.carbondesignsystem.com/?path=/story/components-form-group--with-ai-label)
with the `revert` control set to true or take a look at the example story
[source code](https://github.com/carbon-design-system/carbon/blob/main/packages/web-components/src/components/form-group/form-group.stories.ts).

## `<cds-ai-label>` attributes, properties and events

Note: For `boolean` attributes, `true` means simply setting the attribute (e.g.
`<cds-ai-label revert-active>`) and `false` means not setting the attribute
(e.g. `<cds-ai-label>` without `revertActive` attribute).
