## Side nav

Side nav in UI Shell consists of the following components:

| Tag                    | Description                                      |
| ---------------------- | ------------------------------------------------ |
| `<side-nav>`           | The top-level container                          |
| `<side-nav-item>`      | The container of the nav items                   |
| `<side-nav-link>`      | Nav item, working as a link                      |
| `<side-nav-menu>`      | Nav item, working as a container of sub-items    |
| `<side-nav-menu-item>` | Nav item, working as as a sub-item and as a link |

### Expanding/collapsing

`<cds-side-nav>` has `expanded` attribute to make it expanded. In its collapsed
state, hovering over the side nav expands it temporarily.

`<cds-side-nav>` can be made non-collapsible, by setting `fixed` attribute.

### Selecting a nav item

Unlike `<cds-dropdown>`, etc., user gesture of side nav item won't update the
selection by itself. The assumption is that following the link in
`<side-nav-link>`/`<side-nav-menu-item>` triggers routing in application, and
application chooses which `<side-nav-link>`/`<side-nav-menu-item>` to set
`active` attribute to, with the new URL.

### Using with header nav

If `<cds-side-nav>` is used along with header (`<cds-header>`), the side nav
should be pushed down. To do that, add below style to your application:

```css
cds-header ~ cds-side-nav {
  margin-top: 3rem;
  height: calc(100% - 3rem);
}
```

## Header nav

Header nav in UI Shell consists of the following components:

| Tag                        | Description                                   |
| -------------------------- | --------------------------------------------- |
| `<cds-header>`             | The top-level container                       |
| `<cds-header-nav>`         | The nav bar                                   |

| `<cds-header-menu-button>` | The expando button for side nav               |
