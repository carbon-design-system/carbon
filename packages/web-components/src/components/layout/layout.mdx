## Overview

The `cds-layout` component provides a way to set layout contexts for specific
parts of a web application. It uses Carbon's experimental `layout` sass module
(exported from `@carbon/styles`) to control layout-related settings like size
and density of all components within a layout context.

All children components that support it will react to the `size` and `density`
attributes you pass to `cds-layout`. Note that not all components support the
entire spectrum of options available. In these cases, a component will typically
cap out at the maximum or minimum size it supports in order to still best suit
the layout context's intent.

If a component is outside a layout context or `size` / `density` is not set, it
will fall back to its default rendering.

Components that are explicitly passed a `size` attribute are unaffected by the
layout context. Example:
```html
<div>
  <!-- This text input is always lg -->
  <cds-text-input size="lg" label="Text input"></cds-text-input>

  <!-- This text input will render with its default size (md) -->
  <cds-text-input label="Text input"></cds-text-input>

  <cds-layout size="sm">
    <!-- This text input will still be lg -->
    <cds-text-input size="lg" label="Text input"></cds-text-input>

    <!-- This text input will be sm -->
    <cds-text-input label="Text input"></cds-text-input>
  </cds-layout>
</div>
```

### cds-layout-constraint

In order to apply specific constraints to children components that might differ
from their own preference, the `cds-layout-constraint` utility component can be
used.

An example of this use case is the icon-only content switcher. It uses
`cds-icon-button` under the hood which supports all sizes and defaults to `lg`.
The `cds-content-switcher` though only supports `sm`->`lg` and defaults to `md`.
```html
<div>
  <cds-layout-constraint size-default="md" size-min="sm" size-max="lg">
    <!-- cds-icon-button is now locked to the sizes sm->lg and defaults to md -->
    <cds-icon-button></cds-icon-button>
  </cds-layout-constraint>
</div>
```

The constraints for size must be passed as separate attributes:

- `size-min`
- `size-default`
- `size-max`

Example: `size-min="sm" size-max="lg"`
